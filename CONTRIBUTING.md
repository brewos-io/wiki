# Contributing to BrewOS Wiki

Thank you for your interest in contributing to the BrewOS Wiki! This document provides guidelines for contributing documentation improvements.

## How to Contribute

### Reporting Issues

If you find errors, unclear sections, or missing information:

1. **Check existing issues** - Search for similar issues first
2. **Create a new issue** - Describe the problem clearly
3. **Provide context** - Include page links and specific sections

### Improving Documentation

1. **Fork the repository**
2. **Create a branch** - Use descriptive branch names
3. **Make your changes** - Follow the style guide below
4. **Test your changes** - Verify links and formatting
5. **Submit a pull request** - Describe your changes

## Style Guide

### Writing Style

- **User-friendly language** - Write for end users, avoid jargon
- **Clear and concise** - Be direct and to the point
- **Step-by-step** - Break complex tasks into steps
- **Examples** - Include practical examples where helpful

### Markdown Formatting

- **Headers** - Use proper heading hierarchy
- **Lists** - Use consistent list formatting
- **Code blocks** - Use appropriate language tags
- **Links** - Use relative links for internal pages
- **Tables** - Format tables consistently

### Content Organization

- **Logical flow** - Organize content logically
- **Cross-references** - Link to related pages
- **Navigation** - Update README files with new content
- **Consistency** - Follow existing patterns

## Types of Contributions

### Documentation Improvements

- Fix typos and grammar errors
- Clarify unclear sections
- Add missing information
- Improve examples
- Update outdated content

### New Content

- Add new guides or tutorials
- Document new features
- Add troubleshooting solutions
- Create machine-specific guides

### Translations

- Translate pages to other languages
- Maintain language-specific directories
- Keep translations up to date

## Pull Request Process

1. **Fork and clone** the repository
2. **Create a branch** for your changes
3. **Make your changes** following the style guide
4. **Test your changes** - Verify formatting and links
5. **Commit** with clear commit messages
6. **Push** to your fork
7. **Create pull request** with description

### Pull Request Guidelines

- **Clear title** - Describe what you changed
- **Detailed description** - Explain why and what
- **Reference issues** - Link to related issues
- **Screenshots** - Include if helpful
- **Test results** - Describe what you tested

## Review Process

- All contributions are reviewed
- Feedback may be requested
- Changes may be suggested
- Approval required before merging

## Questions?

If you have questions about contributing:

- **Check existing issues** - Your question may be answered
- **Create an issue** - Ask for clarification
- **GitHub Discussions** - Join community discussions
